## How to read a file directory?

Surely, there are more than one way of approaching this. One way that I did is by using Node's `fs` module. We can do this since we are using a <LinkInPage text='React Server Component' href='https://react.dev/reference/rsc/server-components' /> .

According to the React docs,

> Server Components are a new type of Component that renders ahead of time, before bundling, in an environment separate from your client app or SSR server.

Now, to read a specific file directory:

```tsx title="files.ts" showLineNumbers
'use server';

import fs from 'fs';
import path from 'path';

export async function getFilesOfPath(...paths: string[]) {
  const DIR = path.join(process.cwd(), ...paths);

  const fileList = await new Promise<fs.Dirent[]>((resolve, reject) => {
    fs.readdir(DIR, { encoding: 'utf8', withFileTypes: true }, (err, files) => {
      if (err) reject(err);

      resolve(files);
    });
  }).then((files) => files);

  return fileList;
}
```

Just pass in an existing, valid directory name and that server-side function will give us the directories under the path that we have provided. For example:

```tsx showLineNumbers
const fileList = await getFilesOfPath('app');
```

## Creating a `<FileNode />` Component

Next, we create the `<FileNode />` which is responsibel for rendering each file directory.

```tsx title="file-node.tsx" showLineNumbers
import { type Dirent } from 'fs';
import { FileIcon, FolderIcon } from 'lucide-react';

export function FileNode({ file }: { file: Dirent }) {
  const isDirectory = file.isDirectory();

  return (
    <li>
      <div className="flex items-center gap-3 px-3 py-2 rounded">
        {isDirectory ? (
          <FolderIcon className="size-4 text-amber-500" />
        ) : (
          <FileIcon className="size-4 text-sky-300" />
        )}
        <span>{file.name}</span>
      </div>
    </li>
  );
}
```

- It receives a `file` prop which is of type `Dirent`;
- It renders a Folder icon besides the name if the file is a directory, a File icon is rendered otherwise.

So then, we can replace the `li` in the `FileList` component with the `FileNode` component:

```tsx title="file-list.tsx" showLineNumbers {1,7}
import { FileNode } from './file-node';

export function FileList({ fileList }: { fileList: Dirent[] }) {
  return (
    <ul className={cn('pl-6 select-none overflow-hidden text-sm')}>
      {fileList.map((file) => (
        <FileNode key={file.path} file={file} />
      ))}
    </ul>
  );
}
```

## Add Toggle behavior if FileNode is a directory

We aim to bring in a toggling functionality but we will implement it without JavaScript. We will leverage the power of TailwindCSS and HTML label and input:checkbox elements. So, if the file is a directory, we render a label within which is a checkbox.

```tsx title="file-node.tsx" showLineNumbers
import { type Dirent } from 'fs';
import { FileIcon, FolderIcon } from 'lucide-react';

export function FileNode({ file }: { file: Dirent }) {
  const isDirectory = file.isDirectory();

  if (isDirectory) {
    const nodeId = [file.path, file.name].join('-');

    return (
      <li>
        <label
          htmlFor={nodeId}
          className="flex items-center gap-2 px-3 py-2 rounded hover:bg-gray-800 cursor-pointer"
        >
          <FolderIcon className="size-4 text-amber-500" />
          <span>{file.name}</span>
          <input
            defaultChecked={false}
            type="checkbox"
            name={nodeId}
            id={nodeId}
            hidden
          />
        </label>
      </li>
    );
  }

  return (
    <li>
      <div className="flex items-center gap-3 px-3 py-2 rounded">
        <FileIcon className="size-4 text-sky-300" />
        <span>{file.name}</span>
      </div>
    </li>
  );
}
```

Note:

- the `nodeId` in line 8 is defined to make sure the id of the checkbox is unique since we can have directories with the same file name.
- the `htmlFor` of the label element should point to the value of the `id` of the checkbox element.
- the checkbox has a `hidden` attribute so it won't be visible.

Let us leave this for now. We will get back to this toggle behavior later.

## Render nested directories and/or files (Recursion!)

Our implementation so far only renders the first-level directories and files relative to the root path we provided ('components', in this case). To be able to render the nested directories and files, first we use `getFilesOfPath` again and then render a `FileList` next to the label element.

```tsx title="file-node.tsx" showLineNumbers {3,5,11,13,31}
import { type Dirent } from 'fs';
import { FileIcon, FolderIcon } from 'lucide-react';
import { FileList } from './file-list';

export async function FileNode({ file }: { file: Dirent }) {
  const isDirectory = file.isDirectory();

  if (isDirectory) {
    const nodeId = [file.path, file.name].join('-');

    const filePath = file.path.replace(process.cwd(), '');

    const fileList = await getFilesOfPath(filePath, file.name);

    return (
      <li>
        <label
          htmlFor={nodeId}
          className="flex items-center gap-2 px-3 py-2 rounded hover:bg-gray-800 cursor-pointer"
        >
          <FolderIcon className="size-4 text-amber-500" />
          <span>{file.name}</span>
          <input
            defaultChecked={false}
            type="checkbox"
            name={nodeId}
            id={nodeId}
            hidden
          />
        </label>
        <FileList fileList={fileList} />
      </li>
    );
  }

  return (
    <li>
      <div className="flex items-center gap-3 px-3 py-2 rounded">
        <FileIcon className="size-4 text-sky-300" />
        <span>{file.name}</span>
      </div>
    </li>
  );
}
```

- In line 5, we turn the FileNode component into a Server Component by making the function `async`;
- Line 11 is needed to define the proper directory to be passed to the `getFilesOfPath` function;
- In Line 31, FileList will now render the nested directories and files in a recursive manner.

## Enter TailwindCSS `peer` and `has`

Now, going back to the toggle behavior. The first step is to add TailwindCSS' `peer` class to the label element:

```tsx showLineNumbers
<label className="peer"></label>
```

By this change we can style its `FileList` sibling (which has a `ul` as the root element) based on the state of the checkbox within the label.

```tsx title="file-list.tsx" showLineNumbers {8}
import { cn } from '@lib/utils';
import { type Dirent } from 'fs';
import { FileNode } from './file-node';

export function FileList({ fileList }: { fileList: Dirent[] }) {
  return (
    // other classes are omitted for brevity
    <ul className={cn('hidden peer-has-[:checked]:block')}>
      {fileList.map((file) => (
        <FileNode key={file.path} file={file} />
      ))}
    </ul>
  );
}
```

By default the nested `<FileList />` is `hidden` but if its peer, which means its label sibling, _has_ a checkbox in it that is _checked_, then we display it with the class `block`.

## Enter TailwindCSS `group`

Next, let us render an open folder icon if the directory is toggled "open".

```tsx showLineNumbers {1-3}
<label htmlFor={nodeId} className="peer group">
  <FolderIcon className="size-4 text-amber-500 group-has-[:checked]:hidden" />
  <FolderOpenIcon className="size-4 text-amber-500 hidden group-has-[:checked]:inline-block" />
  <span>{file.name}</span>
  <input
    defaultChecked={false}
    type="checkbox"
    name={nodeId}
    id={nodeId}
    hidden
  />
</label>
```

- We add the `group` class to the label element;
- We render a `FolderIcon` but when its parent which is the label has a checked descendant, the `hidden` class will be applied to it to hide it;
- We render a `FolderOpenIcon` which is hidden by default but when its parent which is the label has a checked descendant, the `inline-block` class will be applied to it to display it;
